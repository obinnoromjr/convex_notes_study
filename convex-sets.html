Convex Sets

The destination of §2.1. We keep everything about an affine set — and change exactly one thing.

segment, not line θ ≥ 0 mastery: design construct & verify
the whole idea → An affine set is closed under the whole line. A convex set is closed only under the segment. The single new rule that does this: the weights must be nonnegative.
00 — orientation

Where we are

§2.1.2 defined the affine set: closed under the whole line through any two of its points. §2.1.4 relaxes one thing — keep only the segment, not the whole line. Equivalently: require the weights to be nonnegative as well as summing to one. That relaxation is the entire content of the word “convex.”

the = vs ≤ flag, in geometry Affine lived under “any scalars, weights sum to 1.” Convex adds θ ≥ 0. Same algebra as linear (=, any scalars) vs convex (≤, convex combination) back in §1.1.
01 — definition

The segment, drawn

\(C\) is convex if the line segment between any two of its points lies in \(C\): for any \(x_1,x_2\in C\) and any \(\theta\) with \(0\le\theta\le1\),

$$\theta x_1 + (1-\theta)x_2 \in C.$$
θ = 0.62 · point = 0.62·x₁ + 0.38·x₂

Drag θ across [0,1] and the yellow point stays inside C. Press the button to let θ run over all of ℝ — the dashed violet line shoots past the endpoints. That extra reach is exactly what convexity drops.

affine — the whole line
θx₁ + (1−θ)x₂ ∈ C
for ALL θ ∈ ℝ
weights sum to 1, any sign
convex — only the segment
θx₁ + (1−θ)x₂ ∈ C
for 0 ≤ θ ≤ 1
weights sum to 1 AND θ ≥ 0 ← the lone new requirement

The weights still sum to one in both columns. The only difference is nonnegativity of \(\theta\) (and therefore of \(1-\theta\)).

02 — intuition & hierarchy

“Every point can see every other point”

A set is convex when every point can be seen by every other point along an unobstructed straight path that stays in the set. Convex sets have no notches, no holes, no dents to hide behind.

Disk — every chord inside. ✓

Crescent — chord exits. refuted

Star — chord crosses the notch. refuted

to prove convex Take arbitrary \(x_1,x_2\in C\) and arbitrary \(\theta\in[0,1]\); show \(\theta x_1+(1-\theta)x_2\in C\). It must hold for every pair.
to refute convex Exhibit one bad segment — a single pair whose connector leaves \(C\). One counterexample is enough.

The hierarchy: each step relaxes what the set must contain

Every affine set is convex — it holds the whole line, so it certainly holds the segment. The converse fails. Careful: a convex set is not “an affine set” — it is a weaker, larger object.

subspace all linear combinations — any weights, no sum constraint, must contain the origin.
affine set \(\sum\theta_i=1\), any sign — the whole line through any two points. ↑ relax: drop “contains 0” / free the sum
convex set \(\sum\theta_i=1\) AND \(\theta_i\ge0\) — only the segment. ↑ relax further: add θ ≥ 0, keep only the segment

subspace ⊆ affine set ⊆ convex set

03 — convex combinations

From a segment to a mixture

A convex combination of \(x_1,\dots,x_k\) is

$$\theta_1 x_1 + \cdots + \theta_k x_k, \qquad \theta_1+\cdots+\theta_k = 1,\quad \theta_i \ge 0.$$

That is: an affine combination plus nonnegativity. (This is a statement about the combination, not about a set.) Two ways to read it:

Geometric reading

The combination lands in the filled region the points span: a segment (2 points), a triangle (3), a polytope (more).

Probabilistic reading

A mixture / weighted average: \(\theta_i\) is the fraction of \(x_i\). The weights are a probability distribution over the points.

θ = (0.33, 0.34, 0.33) · Σ = 1.00 drag the yellow point — the bars always sum to 1, and the point never leaves the triangle
the equivalence A set is convex ⟺ it contains every convex combination of its points. The segment definition (\(k=2\)) already implies the general case.
04 — convex hull

The smallest convex set around a cloud

The convex hull \(\operatorname{conv}C\) is the set of all convex combinations of points of \(C\):

$$\operatorname{conv}C = \{\,\theta_1x_1+\cdots+\theta_kx_k \mid x_i\in C,\ \theta_i\ge0,\ \textstyle\sum_i\theta_i=1\,\}.$$
the hull snaps to the outermost points; everything else sits inside the filled region
minimality flag \(\operatorname{conv}C\) is always convex, and it is the smallest convex set containing \(C\): if \(B\) is any convex set with \(C\subseteq B\), then \(\operatorname{conv}C\subseteq B\). It sits inside every convex superset.

Hull vs hull

conv of 2 points = the segment. conv of 3 non-collinear points = the filled triangle. The hull is the filled-in region.

Contrast: affine hull

aff of those same points = the line / the whole plane. Same points — the convex hull fills in, the affine hull is the carrying flat that runs off to infinity.

05 — generalizing to expectation

Closed under averaging

Convex combinations extend to infinite sums, to integrals, and — most generally — to expectation. If \(C\) is convex and \(x\) is a random vector with \(x\in C\) almost surely (and \(\mathbb{E}x\) exists), then

$$\mathbb{E}\,x \in C.$$

Each grey dot is a possible value of x (size = probability). Every value sits in C, so the weighted mean E x — a giant convex combination — is dragged inside too.

the two-point case is just the segment Let \(x=x_1\) with probability \(\theta\) and \(x=x_2\) with probability \(1-\theta\). Then \(\mathbb{E}x=\theta x_1+(1-\theta)x_2\) — the plain convex combination. The probabilities are the weights; “probabilities \(\ge0\)” is the nonnegativity.

A convex set is closed under averaging. This is the fact sitting behind Jensen’s inequality (Ch. 3, forward).

06 — expert lens (design)

How to actually use this

The mastery target here is design: you’re asked to construct and verify, not just recognize. Four moves carry that:

  • Prove convex via the segment test (arbitrary pair, arbitrary θ); refute with one bad segment.
  • Certify a point lies in a convex set by exhibiting it as a convex combination of known members — weights \(\ge0\), summing to 1. This is the design move.
  • \(\operatorname{conv}C\) is the smallest convex set containing \(C\) — it sits inside every convex superset.
  • A convex combination is an affine combination + nonnegativity — at the combination level. A convex set is not an affine set.
