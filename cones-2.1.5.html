Cones.

The third sibling of affine and convex. Keep nonnegativity, drop the sum-to-one — and a flat set folds up into a pie slice with its apex pinned at the origin.

Mastery · diagnose closes §2.1 affine · convex · conic
00

Where we are

three sections, three weight rules

Section 2.1 builds sets out of combinations of points. Each subsection keeps the same machinery — a weighted sum \(\theta_1x_1+\dots+\theta_kx_k\) — and changes only the rule on the weights.

§2.1.2 gave the affine combination (weights sum to 1). §2.1.4 gave the convex combination (sum to 1 and nonnegative). §2.1.5 adds the last variant — the conic combination — by keeping nonnegativity and dropping the sum-to-one. The set it generates is the convex cone.

01

Cone

L1360–1361 · nonnegative homogeneous

\(C\) is a cone (or nonnegative homogeneous) if for every \(x\in C\) and \(\theta\ge 0\), \(\theta x\in C\).

A cone is closed under scaling by any nonnegative factor. Take a point, stretch it or shrink it toward the origin by any \(\theta\ge 0\), and you never leave the set. Geometrically: if a point is in, the entire ray from the origin through it is in.

Fast check — read 1 A nonempty cone contains 0 (take \(\theta=0\): \(0\cdot x = 0\)). The origin is the apex. So a set that misses the origin cannot be a cone.
Trap — read 2 A cone need not be convex. “Cone” names only the scaling property. The union of two rays is a cone — every point scales along its ray — yet it fails convexity.
02

Convex cone

L1361–1367 · the pie slice

\(C\) is a convex cone if it is convex AND a cone \(\iff\) for any \(x_1,x_2\in C\) and \(\theta_1,\theta_2\ge 0\), \(\theta_1x_1+\theta_2x_2\in C\).

That single condition packs both properties. It is the workhorse test for this section — read it two ways:

set x₁ = x₂→ θx ∈ C  (cone)
set θ₁+θ₂ = 1→ segment ∈ C  (convex)

Geometrically (B&V Fig 2.4) a convex cone is the pie slice: apex at \(0\), two edges running through \(x_1\) and \(x_2\), and every direction between them filled in. The nonnegative orthant \(\mathbf{R}_+^n\) is the prototype.

Drag the weights below. Whatever nonnegative \(\theta_1,\theta_2\) you pick, the combination \(\theta_1x_1+\theta_2x_2\) stays inside the wedge — that is the convex-cone property, live.

03

Conic combinations

L1368–1373 · the whole section in one row

\(\theta_1x_1+\dots+\theta_kx_k\) with \(\theta_i\ge 0\) is a conic combination (nonnegative linear combination).

Line up all three rules and the pattern is exact: conic combination = drop sum-to-one, keep nonnegativity.

combinationΣθᵢ = 1 ?θᵢ ≥ 0 ?net rule
affinerequired—Σθ = 1 (any sign)
convexrequiredrequiredΣθ = 1 & θ ≥ 0
conicdroppedrequiredθ ≥ 0 only
The defining equivalence \(C\) is a convex cone \(\iff\) it contains every conic combination of its elements. (Mirror it: convex set ↔ contains every convex combination; subspace ↔ every linear combination.)
04

Conic hull

L1374–1378 · the smallest convex cone

The conic hull of \(C=\{\theta_1x_1+\dots+\theta_kx_k \mid x_i\in C,\ \theta_i\ge 0\}\) = the smallest convex cone containing \(C\).

Take any set \(C\), allow every nonnegative weighting of its points, and you sweep out the tightest wedge that wraps it — the conic hull. It is the cone analogue of \(\operatorname{aff} C\) and \(\operatorname{conv} C\).

Stacked against its siblings, each “smallest set of its kind” is just the previous rule with one constraint flipped:

The relationship, compressed convex = affine + nonnegativity.   conic = convex with the sum-to-one dropped. Each hull is the smallest set of its type ( \(\subseteq\) every set of that type containing \(C\) ).
05

Expert lens

mastery target · diagnose

At the diagnose level you place an unknown set in the trilogy by reading off which weight constraints it enforces, and you prove or refute each property with the right one-line test.

Is it a cone?

Test nonnegative homogeneity: \(\theta x\in C\) for all \(\theta\ge0\) — independent of convexity. Quick refute: a nonempty cone must hold \(0\); a set off the origin isn't one.

Is it a convex cone?

One test does it: \(\theta_1x_1+\theta_2x_2\in C\) for \(\theta_i\ge0\). Prove with that line; refute convexity-of-a-cone with a bad segment (union of two rays).

Place it in the trilogy

Sort by enforced constraints: affine (Σ=1) · convex (Σ=1, ≥0) · conic (≥0). A convex cone is guaranteed closed under conic combos — but not under all linear combos. Subspaces are the special case that are.

Build the smallest one

conic hull = smallest convex cone containing \(C\) — the exact mirror of \(\operatorname{aff} C\) and \(\operatorname{conv} C\).

Five things to walk away with

  • Coneθx ∈ C for all θ ≥ 0 (nonnegative homogeneous). Nonempty ⟹ contains 0. Need not be convex.
  • Convex coneconvex AND cone ⟺ θ₁x₁+θ₂x₂ ∈ C for θᵢ ≥ 0. The pie slice, apex 0.
  • Conic comboθᵢ ≥ 0, no sum-to-one (= nonnegative linear combination). Convex cone ⟺ contains all of them.
  • Conic hullthe smallest convex cone containing C — mirror of aff C / conv C.
  • The trilogyaffine (Σ=1) · convex (Σ=1, ≥0) · conic (≥0). Convex = affine + nonnegativity; conic = convex minus sum-to-one. Hulls: span / aff / conv / conic hull.
